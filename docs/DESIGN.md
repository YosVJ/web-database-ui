# DESIGN SYSTEM

## Product
Internal ERP system for web and mobile-responsive use.

## Design Goal
Create a clean, modern, enterprise-grade interface that is highly readable, efficient, and professional.

## Design Tone
- Modern
- Minimal but not empty
- Professional (B2B)
- Clean and structured
- High readability

## Platforms
- Desktop
- Tablet
- Mobile (responsive)

## Core Screens
- Login
- Dashboard
- Users
- Purchase Requests
- Approval Queue
- Reports

## Layout
- Sidebar navigation (desktop)
- Top bar header
- Card-based dashboard
- Table-based data views
- Forms and modals

## Components
- KPI Cards
- Tables
- Forms
- Buttons
- Status badges
- Search and filters
- Modal dialogs
- Loading states
- Empty states

## UI Rules
- Clean spacing
- Clear hierarchy
- Avoid clutter
- Tables must be readable
- Forms must be simple

## Color Rules
- Light mode must feel modern and alive
- Dark mode must remain clean
- Use colors for status:
  - Pending
  - Approved
  - Rejected

## Mobile Rules
- Do not shrink desktop UI blindly
- Stack components properly
- Maintain usability for forms

## Constraints
- No overdesign
- No unnecessary animation
- No cluttered UI
- Focus on usability first
